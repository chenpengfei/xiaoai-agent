# 2.4 步骤：端到端验证和运维检查

## 目标

验证从音箱到 Mac Mini server 的闭环：

```text
LX06 client -> Mac Mini server -> server 指令 -> 音箱播报
```

当前已经验证：server 下发测试指令时，音箱会播报“已连接”。

## Mac Mini 侧检查

```sh
lsof -nP -iTCP:4399 -sTCP:LISTEN
docker ps
```

## 音箱侧检查

```sh
ssh -o HostKeyAlgorithms=+ssh-rsa root@192.168.1.2

ps | grep open-xiaoai
netstat -tn | grep 192.168.1.9:4399
tail -f /data/open-xiaoai/client.log
```

成功连接时，`client.log` 中会出现：

```text
✅ 已连接: "ws://192.168.1.9:4399"
```

server 下发测试指令时，音箱会播报：

```text
已连接
```

## 手动冷启动验证

Agent 环境不能直接执行远程 `reboot`，因为 reboot/shutdown 类命令会被安全策略拦截。

手动验证方法：

1. 拔掉音箱电源。
2. 等 10-20 秒。
3. 重新插电。
4. 等 2-3 分钟。
5. 在 Mac Mini 上验证：

```sh
nc -vz 192.168.1.2 22
lsof -nP -iTCP:4399 -sTCP:LISTEN
```

然后 SSH 进入音箱：

```sh
ssh -o HostKeyAlgorithms=+ssh-rsa root@192.168.1.2
```

检查 client 是否自启动并连接 Mac Mini：

```sh
ps | grep open-xiaoai
netstat -tn | grep 192.168.1.9:4399
tail -40 /data/open-xiaoai/client.log
```

## 常见排障

### 22 端口 refused

先确认 IP 是真实音箱：

```sh
arp -an
nc -vz 192.168.1.2 22
ssh-keyscan -T 3 -p 22 192.168.1.2
```

当前真实音箱：

```text
IP: 192.168.1.2
MAC: 50:88:11:6f:f2:a8
```

### client 连不上 server

检查：

1. Mac Mini IP 是否仍是 `192.168.1.9`。
2. Mac Mini 是否监听 `4399`。
3. 音箱 `/data/open-xiaoai/server.txt` 是否仍是 `ws://192.168.1.9:4399`。
4. Docker server 是否还在运行。

### IP 变化

如果 DHCP 变更：

- 用 MAC `50:88:11:6f:f2:a8` 找回音箱。
- 如果 Mac Mini IP 改了，更新音箱 `/data/open-xiaoai/server.txt`。
