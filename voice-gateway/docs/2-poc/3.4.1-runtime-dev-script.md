# 3.4.1 步骤：开发期运行脚本和日志

## 目标

在 `3.3` 最小 Hermes 闭环已跑通后，先把开发期启动方式固定下来。

本步骤只解决：

```text
一条命令启动 Mac Mini xiaozhi Hermes server
  -> 固定 PATH
  -> 检查必要依赖
  -> 检查 4399 是否被占用
  -> 输出日志到文件
```

暂不做：

- launchd 开机自启动。
- Docker 化。
- 本地 STT。
- 自定义 TTS。
- 连续对话。

## 运行脚本

脚本位置：

```text
/Users/chenpengfei/projects/vibe-coding/xiaoai-agent/open-xiaoai/scripts/run-xiaozhi-hermes-dev.sh
```

使用方式：

```sh
cd /Users/chenpengfei/projects/vibe-coding/xiaoai-agent/open-xiaoai
./scripts/run-xiaozhi-hermes-dev.sh
```

脚本内部会进入：

```text
/Users/chenpengfei/projects/vibe-coding/xiaoai-agent/open-xiaoai/examples/xiaozhi
```

并执行：

```sh
CLI=true uv run main.py
```

脚本内部会设置：

```sh
PYTHONUNBUFFERED=1
```

这样 Python 输出经过 `tee` 管道时也会实时刷新到控制台和日志文件，避免 `print` 被块缓冲导致控制台暂时看不到日志。

## 日志文件

开发期日志输出到：

```text
/Users/chenpengfei/projects/vibe-coding/xiaoai-agent/logs/xiaozhi-hermes-dev.log
```

看日志：

```sh
tail -f /Users/chenpengfei/projects/vibe-coding/xiaoai-agent/logs/xiaozhi-hermes-dev.log
```

## 脚本会检查什么

1. `uv` 是否存在。
2. `hermes` 是否存在。
3. `xiaozhi/models/silero_vad.onnx` 是否存在。
4. `sherpa_onnx` 所需的 `libonnxruntime.1.23.2.dylib` 链接是否存在，不存在则自动补链接。
5. 端口 `4399` 是否已被占用。

如果 `4399` 已经被占用，脚本会退出并打印占用进程，不会自动 kill，避免误杀当前正在调试的 server。

## 验证方式

启动后检查端口：

```sh
lsof -nP -iTCP:4399 -sTCP:LISTEN
```

音箱侧说：

```text
问问小马 你是谁
```

server 日志应出现：

```text
🔥 收到指令: 问问小马 你是谁
🤖 调用 Hermes: 你是谁
🤖 Hermes 回答: ...
```

## 成功标准

- 通过脚本能稳定启动 server。
- 端口 `4399` 正常监听。
- 日志文件能记录启动信息和后续 server 输出。
- `问问小马 <问题>` 仍能触发 Hermes 回答并让音箱播报。
