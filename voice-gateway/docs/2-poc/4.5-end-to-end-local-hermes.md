# 4.5 步骤：端到端 Hermes 完全接管验证

## 目标

串起完整 4-* 本地语音链路：

```text
音箱本地 KWS / VAD
  -> 音箱 record stream
  -> Mac Mini sherpa-onnx VAD / STT
  -> Hermes Gateway API Server
  -> Mac Mini TTS URL
  -> 音箱播放
```

本步骤完成后，Hermes 不再只是“小米云回答之后的打断者”，而是音箱语音问答的主路径。

## 前置条件

前置条件已经完成：

- `4.2-speaker-kws-vad-audio.md`
  - 能拿到音箱真实语音样本。
  - 能切出一次完整 utterance。

- `4.3-mac-mini-stt.md`
  - 音箱样本可以转文字。
  - `你好 <问题>` 可以解析成 question。

- `4.4-mac-mini-tts-playback.md`
  - Mac Mini 生成的 TTS 音频可以通过 TTS URL 在音箱播放。

如果任一前置条件不满足，不要进入端到端联调。

## 端到端 session 状态机

建议定义一个清晰状态机：

```text
IDLE
  -> WAKE_DETECTED
  -> RECORDING
  -> SPEECH_ENDED
  -> TRANSCRIBING
  -> ASKING_HERMES
  -> SYNTHESIZING
  -> PLAYING
  -> IDLE
```

失败状态：

```text
STT_FAILED
HERMES_FAILED
TTS_FAILED
PLAYBACK_FAILED
```

失败后回到：

```text
IDLE
```

不要让失败 session 卡住后续唤醒。

## 统一日志格式

每个请求必须有 `session_id`。

日志示例：

```text
🎙️ session=20260517-001 wake keyword=你好
🎙️ session=20260517-001 vad_start
🎙️ session=20260517-001 vad_end audio_ms=2140 path=/.../utterances/001.wav
📝 session=20260517-001 stt text='你好今天适合去哪玩' duration_ms=860
🤖 session=20260517-001 hermes question='今天适合去哪玩' duration_ms=2100
🔊 session=20260517-001 tts audio=/.../tts/001.wav duration_ms=720
📢 session=20260517-001 playback_start total_ms=3920
✅ session=20260517-001 done total_ms=8800
```

## 和 3-* 路线的对比

联调时同时记录 3-* 和 4-* 的体验差异。

| 指标 | 3-* legacy route：小米云 ASR + Hermes 打断 | 4-* 主链路：本地 KWS/VAD + STT/TTS |
|---|---|---|
| ASR 来源 | 小米云 | Mac Mini STT |
| TTS 来源 | 音箱/小米 TTS | Mac Mini TTS URL |
| 是否需要打断小米云 | 是 | 理想情况下否 |
| Hermes 是否主控 | 部分接管 | 完全接管 |
| 延迟 | 受小米云和打断影响 | 可优化 |
| 可控性 | 中 | 高 |

## 测试用例

### 用例 1：短问答

用户说：

```text
你好你是谁
```

期望：

```text
STT question = 你是谁
Hermes 返回身份说明
音箱通过 TTS URL 播放 Hermes 回答
```

### 用例 2：生活问题

用户说：

```text
你好今天适合带孩子去哪玩
```

期望：

```text
回答短、口语化、适合播报
```

### 用例 3：空问题

用户只说：

```text
你好
```

期望：

```text
音箱提示：你想问什么？
不调用 Hermes 或调用极短 fallback
```

### 用例 4：长一点的问题

用户说：

```text
你好帮我解释一下大智若愚是什么意思
```

期望：

```text
STT 核心语义准确
Hermes 回答不超过 200 字
TTS 播放不中断
```

### 用例 5：连续两次请求

用户先说：

```text
你好你是谁
```

播放完成后再说：

```text
你好讲一个十秒钟的小故事
```

期望：

```text
两个 session 都能完整结束
没有 stale connection
没有旧音频串到新 session
```

## 打断策略

4-* 主链路不需要打断小米云，因为主链路不走小米云 ASR fallback。

但实际验证时可能仍需要处理：

1. 小爱原生也被“你好”唤醒。
2. 小米云开始回答。
3. Hermes TTS 正在播放时用户再次说话。

策略建议：

```text
legacy route：保留 speaker.abort_xiaoai()。
4-* 主链路：优先修本地链路，不把小米云 ASR 做成 runtime fallback。
```

Hermes 播放期间的用户打断，后续可以单独开 `4.6` 或 `5-*` 处理，不要阻塞 4.5。

## 成功标准

4.5 完成标准：

- [x] 用户说 `你好 <问题>` 后，不依赖小米云文本也能得到 Hermes 回答。
- [x] 音箱本地 KWS / VAD 能提供本地语音入口。
- [x] Mac Mini sherpa-onnx VAD / STT 能输出可用文本。
- [x] Hermes 能返回短回答。
- [x] Mac Mini TTS URL 音频能在音箱播放。
- [x] 每个 session 都有完整耗时日志。
- [x] 连续请求没有 server 卡死、旧连接残留或状态串扰。
- [x] 失败后能回到 IDLE。
- [x] 3-* legacy route 仍可启动，并作为 manual rollback 保留。
