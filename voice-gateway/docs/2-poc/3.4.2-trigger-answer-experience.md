# 3.4.2 步骤：触发词和回答体验优化

## 目标

在 `3.4.1` 开发期运行脚本验证正常后，继续优化最小闭环的使用体验。

本阶段仍然不引入本地 STT、自定义 TTS、连续对话或开机自启动，只优化当前已跑通的链路：

```text
小爱原生识别文本
  -> 小马触发词
  -> Hermes CLI
  -> 短回答
  -> 音箱自身播报
```

## 当前触发词

主触发词：

```text
问问小马 <问题>
```

额外兼容：

```text
问小马 <问题>
小马小马 <问题>
```

示例：

```text
问问小马 你是谁
问小马 今天晚饭吃什么
小马小马 解释一下黑洞是什么
```

## 代码位置

文件：

```text
https://github.com/idootop/open-xiaoai/tree/main/examples/xiaozhi/config.py
```

新增常量：

```python
HERMES_TRIGGER_PREFIXES = (
    "问问小马",
    "问小马",
    "小马小马",
)
```

新增函数：

```python
def extract_hermes_question(text: str) -> str | None:
```

作用：

- 统一处理触发词。
- 命中触发词时截出问题。
- 自动去掉问题开头的中文/英文逗号、句号、问号、冒号等标点。
- 未命中则返回 `None`，不影响原有小爱和小智逻辑。

## 回答长度策略

Hermes prompt 已从 300 字以内改成 200 字以内：

```text
请用适合语音播报的中文回答，控制在 200 字以内，口语化，不要使用 Markdown，不要输出代码块，不要输出很长列表。
```

server 侧最终 stdout 截断从 600 字改为 400 字：

```python
return answer[:400] or "Hermes 没有返回内容。"
```

这样可以减少音箱长时间播报，也降低回答被中途打断的概率。

## 日志优化

命中触发词后，server 日志现在会输出：

```text
🤖 触发小马: raw='问问小马 你是谁', question='你是谁'
🤖 Hermes 用时: 3.2s
🤖 Hermes 回答: ...
```

这样可以区分：

- 小爱原始识别文本是什么。
- 实际传给 Hermes 的问题是什么。
- Hermes CLI 调用耗时多久。
- Hermes 返回了什么。

## 验证方式

重启 server 后分别测试：

```text
问问小马 你是谁
问小马 今天晚饭吃什么
小马小马 讲一个十秒钟的小故事
```

预期：

- 三个触发词都能进入 Hermes 分支。
- 普通小爱指令不进入 Hermes 分支。
- `召唤小智` 原有逻辑不受影响。
- 控制台实时看到 `🤖 触发小马`、`🤖 Hermes 用时`、`🤖 Hermes 回答`。
- 音箱播报比之前更短。

## 成功标准

- 触发词兼容生效。
- 回答长度适合语音播报。
- 日志能明确显示 raw text、question、耗时和回答。
- 未命中的普通指令不影响原有逻辑。

- 本地 STT。
- 自定义 TTS 音色。
- 连续对话。
- 自然打断。
- launchd 开机自启动。
