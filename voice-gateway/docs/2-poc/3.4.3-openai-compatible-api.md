# 3.4.3 步骤：从 Hermes CLI 改为 Hermes Gateway API Server

## 目标

把当前 `config.py` 中的 Hermes CLI subprocess 调用：

```text
hermes chat -Q -q <prompt>
```

替换为直接调用 Hermes Gateway 暴露的 OpenAI-compatible 接口：

```text
POST http://127.0.0.1:8642/v1/chat/completions
```

这样可以减少每次语音请求启动 Hermes CLI 的开销，也让小爱桥接代码只依赖一个标准 OpenAI-compatible HTTP API。

## Hermes Gateway API Server

Hermes 的 API Server 已在：

```text
/Users/chenpengfei/.hermes/.env
```

启用，本地限定配置如下：

```sh
API_SERVER_ENABLED=true
API_SERVER_HOST=127.0.0.1
API_SERVER_PORT=8642
API_SERVER_MODEL_NAME=hermes-agent
API_SERVER_KEY=<随机生成，保存在 ~/.hermes/.env，不写入项目文档>
```

`API_SERVER_HOST=127.0.0.1` 表示只允许 Mac Mini 本机访问，不对局域网开放。

启动或重启 Gateway 后，OpenAI-compatible base URL 是：

```text
http://127.0.0.1:8642/v1
```

## 当前默认配置

服务侧代码从环境变量读取接口配置：

```sh
XIAOAI_OPENAI_BASE_URL=http://127.0.0.1:8642/v1
XIAOAI_OPENAI_MODEL=hermes-agent
XIAOAI_OPENAI_API_KEY=<默认由启动脚本从 ~/.hermes/.env 的 API_SERVER_KEY 读取>
XIAOAI_OPENAI_TIMEOUT=90
```

如果临时要改成其他 OpenAI-compatible 服务，可以在启动前覆盖这些变量。

## 代码位置

文件：

```text
open-xiaoai/examples/xiaozhi/config.py
```

现在 `ask_xiaoma(question)` 不再调用 Hermes CLI，而是：

```python
requests.post(
    f"{OPENAI_COMPAT_BASE_URL}/chat/completions",
    headers=headers,
    json=payload,
    timeout=OPENAI_COMPAT_TIMEOUT,
)
```

使用 OpenAI Chat Completions 消息格式：

```python
messages = [
    {
        "role": "system",
        "content": "你是小马，一个接在小爱音箱后面的家庭语音助手。回答要简短、自然、适合直接朗读。",
    },
    {"role": "user", "content": user_prompt},
]
```

## 启动脚本

脚本：

```text
open-xiaoai/scripts/run-xiaozhi-hermes-dev.sh
```

会设置默认值：

```sh
export HERMES_ENV_PATH="${HERMES_ENV_PATH:-/Users/chenpengfei/.hermes/.env}"
export XIAOAI_OPENAI_BASE_URL="${XIAOAI_OPENAI_BASE_URL:-http://127.0.0.1:8642/v1}"
export XIAOAI_OPENAI_MODEL="${XIAOAI_OPENAI_MODEL:-hermes-agent}"
export XIAOAI_OPENAI_TIMEOUT="${XIAOAI_OPENAI_TIMEOUT:-90}"
```

如果 `XIAOAI_OPENAI_API_KEY` 没有显式设置，脚本会从 `$HERMES_ENV_PATH` 中读取 `API_SERVER_KEY` 并导出为 `XIAOAI_OPENAI_API_KEY`。

这样项目脚本不用硬编码 token，也不需要手动重复 export API key。

## 启动顺序

先启动 Hermes Gateway：

```sh

```

看到类似日志即可：

再启动 xiaozhi 开发 server：

```sh

```

## 成功标准

- Hermes Gateway API Server 在 `127.0.0.1:8642` 监听。
- API Server 不绑定 `0.0.0.0`，不对局域网开放。
- `config.py` 不再通过 `hermes chat -Q -q` 启动 CLI 子进程。
- 命中触发词后，请求 Hermes Gateway 的 `/v1/chat/completions`。
- 启动脚本能自动从 `~/.hermes/.env` 读取 `API_SERVER_KEY`。
- 音箱仍能播报回答。

## 排障

### connection refused

说明 Hermes Gateway 没启动，或 API Server 没启用。确认：

```sh
lsof -nP -iTCP:8642 -sTCP:LISTEN
```

以及：

```sh
hermes config env-path
```

对应 `.env` 中应有：

```sh
API_SERVER_ENABLED=true
API_SERVER_HOST=127.0.0.1
API_SERVER_PORT=8642
API_SERVER_KEY=...
```

### 401 / Invalid token

说明 `XIAOAI_OPENAI_API_KEY` 和 Hermes `.env` 中的 `API_SERVER_KEY` 不一致。

默认用 `open-xiaoai/scripts/run-xiaozhi-hermes-dev.sh` 启动时，脚本会自动读取，不需要手动设置。

如果手动启动 xiaozhi server，需要自己 export：

```sh
export XIAOAI_OPENAI_API_KEY="$(python3 - <<'PY'
from pathlib import Path
for line in Path('/Users/chenpengfei/.hermes/.env').read_text().splitlines():
    if line.startswith('API_SERVER_KEY='):
        print(line.split('=', 1)[1].strip().strip('"').strip("'"))
        break
PY
)"
```

### model not found

对 Hermes API Server 来说，请求里的 `model` 主要是 OpenAI-compatible 客户端字段。当前默认使用：

```sh
XIAOAI_OPENAI_MODEL=hermes-agent
```

Hermes 实际底层模型仍由 Hermes 自己的配置决定。

### 回答太长

继续调小 `config.py` 里的语音回答 prompt 字数限制，或调低 `max_tokens`。
