## 模块边界

### 音箱侧

音箱侧尽量只做低层能力：

- 麦克风音频采集。
- 本地 KWS。
- 本地 VAD。
- WebSocket 上报音频帧和事件。
- 接收 Mac Mini 下发的播放指令或音频流。
- 扬声器播放。

音箱侧不做：

- Hermes 调用。
- 大模型上下文管理。
- 长文本 TTS 生成。
- 复杂业务逻辑。

### Mac Mini 侧

Mac Mini 负责主逻辑：

- 管理音箱连接。
- 管理语音 session。
- 缓存一次用户说话的音频。
- 使用 sherpa-onnx Silero VAD 做服务端音频切分和保护。
- 使用 sherpa-onnx Paraformer STT 转写问题。
- 调用 Hermes。
- 调用 TTS。
- 生成 TTS URL 并让音箱播放。
- 记录链路日志和耗时。

### Hermes

Hermes 只接收文本问题，返回适合语音播报的短回答：

```text
请用适合语音播报的中文回答，控制在 200 字以内，口语化，不要使用 Markdown。
```

## 数据格式建议

### 音频输入

第一阶段优先保存为文件做离线验证：

```text
/tmp/xiaoai-utterances/YYYYMMDD-HHMMSS-session.wav
```

建议统一到：

```text
format: wav
sample_rate: 16000
channels: 1
sample_width: 16-bit PCM
```

如果音箱传的是 opus 或其他编码，先单独写转换步骤，不要和 STT 绑定在一起。

### STT 输出

统一结构：

```json
{
  "session_id": "20260517-...",
  "text": "你好今天适合去哪玩",
  "language": "zh",
  "duration_ms": 1234,
  "engine": "sherpa-onnx"
}
```

### Hermes 输出

统一结构：

```json
{
  "session_id": "20260517-...",
  "answer": "今天适合去附近的公园...",
  "duration_ms": 2345,
  "model": "hermes-agent"
}
```

### TTS 输出

统一结构：

```json
{
  "session_id": "20260517-...",
  "audio_path": "/tmp/xiaoai-tts/....mp3",
  "duration_ms": 800,
  "engine": "edge-tts",
  "playback": "tts-url"
}
```

## 延迟指标

每次端到端请求都要打印：

```text
session_id=...
kws_ms=...
vad_end_ms=...
stt_ms=...
hermes_ms=...
tts_ms=...
playback_start_ms=...
total_to_first_audio_ms=...
```

第一阶段目标：

```text
STT: < 2s
Hermes: < 5s
TTS: < 2s
总首播: < 7s
```

优化阶段目标：

```text
总首播: < 3-5s
```

## legacy route / manual rollback 策略

4-* 主线不再走小米云 ASR fallback。当前主链路直接使用 Server 端 sherpa-onnx VAD / STT：

```text
音箱 record stream
  -> Mac Mini sherpa-onnx VAD/STT
  -> Hermes
  -> Mac Mini TTS
  -> 音箱播放
```

3-* 小米云 ASR + Hermes 回答只作为 legacy route / manual rollback 保留，不作为 4-* 实时链路里的 runtime fallback。也就是说：验证 4.4/4.5 时，如果 Server 端 STT 失败，应修复本地 STT/VAD/音频切分，而不是自动切回小米云 ASR。

不要一开始就删除或重写：

```text
open-xiaoai/examples/xiaozhi/config.py
open-xiaoai/scripts/run-xiaozhi-hermes-dev.sh
```

建议为 4-* 新增独立脚本：

```text
scripts/run-local-voice-hermes-dev.sh
```

这样 3-* 和 4-* 能并行验证。
