# 4.2 步骤：音箱侧 KWS/VAD 和音频采集验证

## 目标

记录音箱本地语音入口的已验证事实：

```text
音箱麦克风
  -> KWS 或唤醒事件
  -> VAD 开始/结束
  -> 一段完整用户语音音频
  -> 发送到 Mac Mini
  -> Mac Mini 保存样本文件
```

本步骤不接 Hermes、不接 TTS，只验证“能不能稳定拿到用户说话音频”。

## 为什么先做这个

4-* 的成败首先取决于音频入口。

如果没有稳定的本地 KWS/VAD 和音频片段，后面的 Mac Mini sherpa-onnx VAD/STT、Hermes、TTS 都会被错误输入干扰。

本步骤要避免同时调试多个变量：

```text
不要同时改 STT
不要同时改 TTS
不要同时改 Hermes
不要同时改音箱播放
```

只看音频采集。

## 已确认的代码入口

验证过程中重点检查这些文件：

```text
open-xiaoai/packages/client-rust/src/services/audio/record.rs
open-xiaoai/packages/client-rust/src/services/monitor/kws.rs
open-xiaoai/packages/client-rust/src/services/monitor/instruction.rs
open-xiaoai/packages/client-rust/src/services/connect/message.rs
open-xiaoai/examples/xiaozhi/src/server.rs
open-xiaoai/examples/xiaozhi/xiaozhi/xiaoai.py
open-xiaoai/examples/xiaozhi/xiaozhi/services/audio/stream.py
```

已确认和需要持续留意的问题：

1. 当前 client 可以把麦克风音频以 `Stream { tag: "record" }` 形式发到 server。
2. 音箱侧确实具备本地 KWS / VAD 能力。
3. Mac Mini 侧仍使用 sherpa-onnx Silero VAD 做服务端切分和保护。
4. `GlobalStream.input()` / `on_input_data()` 是 Mac Mini 侧承接 record stream 的入口。
5. 音频编码、采样率和分帧方式仍需要在工程化阶段持续记录，避免后续变更破坏 STT。

## 第一阶段：只保存原始音频

验证阶段新增过一个临时开发模式：收到音频后先保存，不做识别。

建议输出目录：

```text
/Users/chenpengfei/projects/vibe-coding/xiaoai-agent/audio-samples/raw/
```

文件命名：

```text
YYYYMMDD-HHMMSS-session-raw.bin
YYYYMMDD-HHMMSS-session.wav
```

如果暂时不知道编码，先保存 `.bin`，并记录：

```text
bytes_len=...
first_16_bytes=...
source=record
timestamp=...
```

## 第二阶段：确认音频格式

用工具检查样本：

```sh
file audio-samples/raw/<sample>.bin
ffprobe audio-samples/raw/<sample>.wav
```

如果是 PCM，需要确认：

```text
sample_rate: 16000 or 24000
channels: 1
sample_width: 16-bit
endianness: little-endian
```

如果是 opus，需要确认是否已有解码逻辑，或者使用 ffmpeg/opuslib 解码到 wav。

目标统一输出：

```text
audio-samples/utterances/<session>.wav
format: 16k mono 16-bit PCM wav
```

## 第三阶段：KWS/VAD 切段

本阶段成功标准不是“能录到声音”，而是“能录到一次完整说话”。

一次 session 需要有：

```text
session_id
wake_detected_at
speech_started_at
speech_ended_at
audio_path
audio_duration_ms
```

日志示例：

```text
🎙️ session=20260517-001 wake_detected keyword=你好
🎙️ session=20260517-001 vad_start
🎙️ session=20260517-001 vad_end duration_ms=2180 audio=/.../utterances/20260517-001.wav
```

## 测试句子

固定用这些短句测试，便于比较：

```text
你好你是谁
你好今天天气怎么样
你好帮我讲一个十秒钟的小故事
你好
```

注意：4-* 的本地路线里，“你好”既可以是 KWS 唤醒词，也可以只是用户完整语音的前缀。这个阶段只需要记录清楚实际识别/录音边界，不急着决定最终产品交互。

## 失败排查

### 没有音频进入 Mac Mini

检查：

```sh
netstat -anv -p tcp | grep '.4399 '
```

确认音箱连接在 `ws://192.168.1.9:4399`。

检查 server 日志是否有 `record` stream。

### 有音频但都是静音

检查：

- 音箱端录音是否真的启动。
- `start_recording` RPC 是否成功。
- PCM 设备是不是 `noop`。
- 当前音箱固件是否允许录音。

### 音频有声音但速度/音调异常

通常是采样率或 sample width 错误。

尝试按不同参数解码：

```text
16k mono s16le
24k mono s16le
48k mono s16le
```

### VAD 结束太慢或太快

先保存完整流，再离线调参数。不要一开始就在实时链路上盲调。
