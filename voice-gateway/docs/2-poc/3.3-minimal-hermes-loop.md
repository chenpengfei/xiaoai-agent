# 3.3 步骤：实现最小 Hermes 闭环

## 目标

在不引入复杂 STT/TTS 的前提下，先跑通最小 Hermes 语音问答闭环。

目标链路：

```text
用户说：问问小马 <问题>
  -> 小爱原生识别文本
  -> xiaozhi config.py 判断触发词
  -> Mac Mini 调用 Hermes CLI
  -> Hermes 返回短回答
  -> speaker.play(text=answer)
  -> LX06 播报回答
```

## 为什么先这样做

当前音箱侧链路已经跑通，但服务侧还没有接入 Hermes。第一版最重要的是验证：Hermes 的回答可以经由 Mac Mini server 传回音箱并播报。

因此第一版先复用：

- 小爱原生 ASR 文本。
- open-xiaoai 当前事件链路。
- 音箱自身文字播报能力。
- Hermes CLI。

暂不做：

- 本地 STT。
- 自定义 TTS 音色。
- 连续对话。
- 自然打断。
- 多轮上下文。

## 建议触发词

```text
问问小马 <问题>
```

例子：

```text
问问小马 今天适合做什么
问问小马 解释一下黑洞是什么
问问小马 帮我想一个晚饭菜单
```

## 修改位置

优先改：

```text
https://github.com/idootop/open-xiaoai/tree/main/examples/xiaozhi/config.py
```

在：

```python
async def before_wakeup(speaker, text, source):
```

里增加一个分支：

```text
如果 source == "xiaoai" 且 text 以 "问问小马" 开头：
  1. 截出问题。
  2. 打断小爱原播报。
  3. 调用 Hermes。
  4. 播放 Hermes 回答。
  5. 不进入后续 xiaozhi 唤醒流程。
```

## 伪代码

```python
import asyncio
import subprocess

async def ask_hermes(question: str) -> str:
    prompt = f"请用适合语音播报的中文回答，控制在 300 字以内，不要使用 Markdown：{question}"
    proc = await asyncio.to_thread(
        subprocess.run,
        ["hermes", "chat", "-Q", "-q", prompt],
        capture_output=True,
        text=True,
        timeout=60,
    )
    if proc.returncode != 0:
        return "Hermes 调用失败，请稍后再试。"
    return proc.stdout.strip() or "Hermes 没有返回内容。"
```

在 `before_wakeup()` 中：

```python
if source == "xiaoai" and text.startswith("问问小马"):
    question = text.removeprefix("问问小马").strip()
    if not question:
        await speaker.play(text="你想问问小马 什么？")
        return False

    await speaker.abort_xiaoai()
    await asyncio.sleep(2)
    answer = await ask_hermes(question)
    await speaker.play(text=answer[:500])
    return False
```

## 成功标准

- 用户说“问问小马 ...”后，server 日志能看到识别文本。
- Hermes CLI 被调用。
- 音箱播报 Hermes 的回答。
- 非 Hermes 触发词不影响原有“召唤小智”示例逻辑。

## 排障入口

如果音箱没反应：

1. 看 Docker / server 日志里有没有 `🔥 收到指令`。
2. 看识别文本是否真的是 `问问小马 ...`。
3. 在 Mac Mini 上单独运行 Hermes CLI。
4. 临时把 `speaker.play(text="测试")` 放到分支里，确认播报可用。
5. 检查 Hermes 命令是否超时或返回非 0。
