# 3.2 步骤：梳理 xiaozhi 示例 server 协议

## 目标

从当前已经跑通的 xiaozhi 示例 server 中，明确服务侧接入 Hermes 需要用到的代码入口：

- WebSocket server 在哪里监听。
- 音箱事件在哪里进入 Python。
- 小爱识别文本在哪里取得。
- 麦克风音频流在哪里取得。
- 如何让音箱播报文本或播放音频。
- 如何打断小爱原有播报。

## 程序入口

```text
https://github.com/idootop/open-xiaoai/tree/main/examples/xiaozhi/main.py
```

入口逻辑：

```text
XiaoAI.setup_mode()
XiaoZhi.instance().run()
```

## WebSocket server 监听位置

```text
https://github.com/idootop/open-xiaoai/tree/main/examples/xiaozhi/src/server.rs
AppServer::run()
addr = "0.0.0.0:4399"
```

连接成功后注册事件和音频流 handler：

```text
MessageHandler::<Event>::instance().set_handler(on_event)
MessageHandler::<Stream>::instance().set_handler(on_stream)
```

## 麦克风音频流入口

```text
src/server.rs:on_stream()
tag == "record" -> PythonManager.call_fn("on_input_data", bytes)
```

Python 侧入口：

```text
xiaozhi/xiaoai.py:on_input_data(data: bytes)
```

当前逻辑会把音频写入：

```text
GlobalStream.input(...)
```

## 小爱识别文本入口

```text
xiaozhi/xiaoai.py:on_event(event: str)
```

关键判断：

```text
SpeechRecognizer / RecognizeResult
payload.results[0].text
is_final == true
```

当前示例会打印：

```text
🔥 收到指令: <text>
```

然后调用：

```text
EventManager.wakeup(text, "xiaoai")
```

## 示例触发逻辑

```text
https://github.com/idootop/open-xiaoai/tree/main/examples/xiaozhi/config.py
before_wakeup(speaker, text, source)
```

当前示例中：

```text
source == "xiaoai" and text == "召唤小智"
```

会执行：

```text
speaker.abort_xiaoai()
speaker.play(text="小智来了，主人有什么吩咐？")
```

## 音箱播报能力入口

```text
xiaozhi/services/speaker.py
SpeakerManager.play(text=...)
```

阻塞文字播报最终调用：

```text
/usr/sbin/tts_play.sh '<text>'
```

非阻塞文字播报最终调用：

```text
ubus call mibrain text_to_speech '{"text": ..., "save": 0}'
```

## 第一版可利用的最小协议

第一版不需要直接处理音频流，可以先用小爱识别文本：

```text
用户说话
  -> 小爱原生 ASR 得到 text
  -> xiaoai.py:on_event()
  -> EventManager.wakeup(text, "xiaoai")
  -> config.py:before_wakeup(speaker, text, source)
  -> 判断 text 是否以 "问问小马" 开头
  -> 调用 Hermes
  -> speaker.play(text=answer)
```

## 成功标准

- server 日志能看到最终识别文本。
- 能区分普通小爱指令和 Hermes 触发词。
- 能调用 `speaker.play(text=...)` 播报任意测试文本。
