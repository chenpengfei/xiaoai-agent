# 2.3 步骤：Mac Mini 上的 open-xiaoai server

## 目标

在 Mac Mini 上运行 open-xiaoai server，监听：

```text
ws://192.168.1.9:4399
```

这个步骤已经完成“示例 server 最小闭环验证”。xiaozhi Docker server 是早期示例，不是最终方案；后续已经基于这条连接链路完成 3-* legacy route 和 4-* 本地 Hermes 接管 PoC。

## 当前已验证状态

已经确认：

- LX06 刷写 `open-xiaoai` 补丁固件正常。
- SSH 可以连接音箱：`root@192.168.1.2`。
- 音箱端 `/data/open-xiaoai/client` 已安装。
- 音箱端 `/data/open-xiaoai/server.txt` 指向 `ws://192.168.1.9:4399`。
- 音箱 client 可以稳定连接 Mac Mini 上的 server。
- Mac Mini 上的 xiaozhi Docker 示例 server 可以监听 `4399`。
- server 下发测试指令时，音箱可以播报测试内容。

## xiaozhi 示例目录

```sh
cd /Users/chenpengfei/projects/vibe-coding/xiaoai-agent/open-xiaoai/examples/xiaozhi
```

## 启动当前 Docker 示例 server

```sh
docker run --rm -p 4399:4399 \
  -v "$PWD/config.py:/app/config.py" \
  idootop/open-xiaoai-xiaozhi:latest
```

说明：

```text
Mac Mini 外部端口 4399 -> 容器内部端口 4399
```

## 查看容器

```sh
docker ps
```

## 查看日志

```sh
docker logs -f $(docker ps --filter ancestor=idootop/open-xiaoai-xiaozhi:latest -q | head -1)
```

## 停止 server

```sh
docker stop $(docker ps --filter ancestor=idootop/open-xiaoai-xiaozhi:latest -q | head -1)
```

## 确认端口监听

```sh
lsof -nP -iTCP:4399 -sTCP:LISTEN
```

## 当前 xiaozhi 示例触发方式

`examples/xiaozhi/config.py` 中默认逻辑包含：

```text
如果小爱识别到 “召唤小智”，server 会打断小爱原回答，并播放 “小智来了，主人有什么吩咐？”
```

配置中的关键词包括：

```text
天猫精灵
小度小度
豆包豆包
你好小智
你好小爱
hi siri
hey siri
```

可优先测试：

```text
召唤小智
```

## 后续演进

当前 server 只是 open-xiaoai 的示例。后续演进路径是：

```text
音箱事件 / 音频 / 识别结果
  -> Mac Mini Hermes server
  -> 判断是否触发 Hermes
  -> STT 或复用小爱识别文本
  -> 调用 Hermes
  -> TTS 或音箱自身播报
  -> 音箱播放回答
```

改造优先级：

1. 3-*：复用小爱原生识别文本接入 Hermes，作为 legacy route / manual rollback。
2. 4-*：使用音箱本地 KWS/VAD、record stream、Mac Mini sherpa-onnx VAD/STT 和 TTS URL 完成本地接管。
3. 3-design：把 PoC 能力收口成稳定的 `voice-gateway`。
